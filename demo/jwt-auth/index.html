03 — Contract System

Every task closes with three contracts.

TASK_CONTRACT
What gets built
Scope, exclusions, tech stack, inputs, and deliverables locked before execution starts. Out-of-scope = blocked. No ROME-style drift.
VERIFY_CONTRACT
When it's done
Binary pass/fail conditions specified as a concrete checklist. Session cannot close until 100% satisfied. No "looks about right."
RESUME_PACK
How to continue
K1 deliverable list + K2 resume prompt + K3 copy-paste code block. Session-expiry resistant. Handoff-ready.
